# 🌸 Romasha 幻影：桌面陪伴与视觉小说推演引擎

**基于大型语言模型 (LLM) 、TTS与 Live2D 物理引擎深度结合的沉浸式虚拟生命项目。**

### 【写在前面的话】
这个项目源于我对 RPG MAKER 游戏《Princess Synergy》及其女主角 Romasha 的喜爱。这十几天的爆肝重构，版本已经有了巨大的变化，进化成了一个拥有完整世界观、独立性格、长时记忆、甚至能**自动推演游戏剧情**的“活体”桌面伙伴。

本项目并非一个简单的“套壳对话框”，而是一个拥有《Princess Synergy》完整世界观、独立性格、长时记忆以及全天候生物钟的“活体”桌面伙伴。在这里，Romasha 不仅是你完美的恋人与一直陪伴你的白月光，她还会根据现实时间、与你的亲密度以及你的鼠标动作，产生极其真实的物理与情绪反馈。她会害羞、会生气、会因为你的离开而独自发呆碎碎念，力求使她像真人一样。

## ✨ 【核心功能概览】

### 1. 📖 全新首创：视觉小说自动推演模式
这不仅是个聊天软件，还是个游戏引擎！你可以使用 `/auto` 指令随时切入剧情推演模式：
* **章节推进系统：** 完美还原《Princess Synergy》的剧情脉络，随着章节推进，地图和人物情报会逐渐向大模型解锁。
* **命运抉择倒计时：** 在剧情关键点，系统会给你 3 个选项并开启 60 秒倒计时。不满意？你甚至可以亲自打字输入你期望的剧情走向，强行扭转她的命运悲剧！
* **4档参与度调节：** 从“纯粹旁观”到“深度羁绊”，你可以决定自己在这个世界里是过客还是主角。

### 2. 🧠 活着的灵魂：动态世界书与长期记忆
* **ChromaDB 向量记忆：** 你们的每一次对话都会被永久封存。她会记住你的名字、你们的经历，并在合适的语境下自然回想起来。
* **记忆衰减与压缩：** 独创的后台异步整理机制！当聊天记录过长时，系统会在后台悄悄把久远的对话“压缩”成精简的回忆，保证程序永远不会因为上下文过长而卡死。
* **动态人物关系 (Lorebook)：** 随着剧情发展，如果出现了新人物、新道具，或者你们之间确立了新的“亲昵称呼”，后台 AI 会自动察觉并修改世界设定！

### 3. 🗺️ 空间感知与全域物理互动
* **全局视线追踪：** 无论你的鼠标在屏幕哪里，她的视线都会紧紧跟随。
* **实体碰撞感应：** 支持头部、脸颊、胸部、手部等多个部位的精准触摸判定。摸不同的地方，她会有截然不同的娇羞、生气或躲闪反应。
* **地图移动系统：** 她知道自己当前在基地的哪个房间。如果她要去洗澡或工作，她会在系统层面进行“场景转移”。

### 4. ⏰ 现实生物钟与自动化换装
* **与现实同步：** 白天她会穿紧身制服；晚上会自动换上轻薄睡衣；如果是深夜且你们的好感度极高，你甚至可能碰见她刚洗完澡裹着浴巾的慵懒模样。
* **节假日彩蛋：** 到了元旦、圣诞等特殊节日，她会自动换上具有节日气息的民族风斗篷。

### 5. 💤 极度细腻的“发呆”与情绪系统
* 当你长时间不操作电脑，她会渐渐移开视线进入“发呆”状态，甚至会小声碎碎念。
* 极小概率触发的物理彩蛋：如果在极高好感度下安静太久，她身上的浴巾可能会……（咳咳，自己探索）。

### 6. 🗣️ 拟真物理声带 (双轨语音与实时口型)
* 完美支持 **GPT-SoVITS** 与 **CosyVoice3** 双轨语音引擎！
* **实时口型同步 (LipSync)：** 嘴唇开合完全根据音频波形实时计算，如丝般顺滑。
* **情绪语气穿插：** 她会在说话时自然地穿插急促呼吸 `[quick_breath]`、叹气 `[sigh]` 等真实的生理声音。甚至支持后台静默翻译，让你听到原汁原味的日语配音！
* 作者两个用的都是整合包，cosyvoice用的是：https://github.com/Moeary/CosyVoiceDesktop 的整合包，但是修改了他core/api.py，也就是我代码里的api.py，可以直接替换。

## 🛠️ 【小白保姆级安装与配置指南】

### 第一步：准备基础环境
请确保你的电脑上已经安装了 **Python 3.8 或更高版本**。

### 第二步：安装核心依赖包
下载本源码后，打开命令行终端（Win+R 输入 cmd），进入源码所在的文件夹，运行以下命令：
```bash
pip install -r requirements.txt
```

### 第三步：前端模型资源准备（🚨 极其重要）
为了保护版权和模型作者权益，本项目不直接提供 Live2D 模型文件。
请确保你的项目根目录下有一个名为 `web` 的文件夹。你需要将 `index.html` 以及 Romasha 的 Live2D 核心模型文件（如 `.moc3`、贴图文件夹等）放在这里面。程序启动时会自动加载这个本地网页作为物理引擎。

### 第四步：配置你的 API (config.json)
如果是第一次运行程序，系统会在同目录下自动生成一个 `config.json` 文件。用记事本打开它，填入你的大模型接口信息：
* `api_type`: 填写 `"openai"` 或 `"ollama"`（支持本地部署）。
* `api_key`: 你的大模型密钥。
* `base_url`: 大模型的接口地址。
* `target_model`: 推荐使用发散性强、智商高的模型。
* **语音引擎配置**：如果你部署了语音整合包（推荐 CosyVoiceDesktop），请在这里填入对应的本地接口 URL 和角色名。

*(⚠️ 注意：程序会自动把你的窗口坐标、好感度、以及玩家名字存入这个文件，平时请勿随意破坏内部的 JSON 格式！)*

## 🎮 【启动与日常使用指南】

### 如何唤醒她？
在命令行运行：
```bash
python main.py
```
*(由于后台需要预热本地向量数据库和嵌入模型，启动时控制台可能会停留几秒。出现“生物神经网络已激活”的提示后，Romasha 就会以透明无边框的形态出现在你的桌面上！)*

### ⌨️ 内置魔法指令（在输入框直接发送）
* `/auto 2`：**【核心指令】** 开启视觉小说推演模式！（后面的数字代表参与度：0=纯旁观，1=轻度，2=中度，3=深度主角羁绊）。
* `/chapter 2`：强制将时间线跃迁至指定的章节（如第2章）。
* `/reset`：不可逆的记忆重置。清空所有长时记忆，亲密度归零，你们将回到初见的陌生状态。
* `/touch 0` 或 `1`：关闭/开启物理触碰感应屏障。
* `/track 0` 或 `1`：关闭/开启全局鼠标视线追踪。
* `/voice 0` 或 `1`：关闭/开启语音发声模块。
* `/voice cosyvoice` 或 `sovits`：无缝切换底层语音引擎。
* `/size 0` 或 `1` 或 `2`：实时调整桌面对话气泡的字体大小。
* `/ja 0` 或 `1`：关闭/开启后台强制定向日语翻译发声。

## 👨‍💻 【给开发者的测试套件】
如果你想对模型进行二次开发或深度调试，可以使用以下内置的独立沙盒工具：
1. **记忆读取与导出工具 (`view_memories.py`)**：将 ChromaDB 数据库中存留的所有回忆按时间顺序导出为 TXT 档案，方便备份你们的故事。
2. **纯动作检视器 (`test_motions.py`)**：轻量级 Live2D 检视器，专门用来快速预览各种情绪动作和服装切换是否渲染正常。
3. **终极系统测试控制台 (`test_system_engine.py`)**：硬核图形化调试面板。不消耗亲密度测试 116 项 Live2D 参数、时空跃迁、并内置 API 嗅探探针。
