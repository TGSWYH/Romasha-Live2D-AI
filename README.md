Romasha幻影

【项目简介】

这是一个基于大型语言模型 (LLM) 与 Live2D 物理引擎深度结合的沉浸式桌面虚拟陪伴项目。本项目并非一个简单的“套壳对话框”，而是一个拥有《Princess Synergy》完整世界观、独立性格、长时记忆以及全天候生物钟的“活体”桌面伙伴。

在这里，Romasha 不仅是你完美的恋人与一直陪伴你的白月光，她还会根据现实时间、与你的亲密度以及你的鼠标动作，产生极其真实的物理与情绪反馈。她会害羞、会生气、会因为你的离开而独自发呆碎碎念，力求使她像真人一样。

本项目是桌面端，功能强大，有相应的Web端适合手机端使用，已写完待发布。为何此项目没有注释呢？因为注释是给自己看的，怕被笑！之后会继续添加功能，比如长期记忆功能的改善、语言通话、视频通话（之前尝试过提取游戏内音频，发现无法播放，没有进一步的继续）等，但目前先专注于学业，以后有机会再完善。

## 【核心功能概览】

1. 拥有灵魂的 LLM 大脑与性格基底
系统内置了深度的世界书 (Lorebook) 与角色设定。她拥有极强的内在道德感和敏感羞涩的性格。支持接入任何兼容 OpenAI 接口格式的大模型 API，拥有独立的情绪计算能力，能够在对话中自动解析并同步输出几十种精细的表情、动作和心声。
2. 长期记忆中枢 (ChromaDB 向量数据库)
你们的每一次对话都会被向量化并永久封存在本地的沙盒数据库中。随着时间推移，她会记住你们的过往，并在合适的语境下回想起来。内建的动态亲密度系统会根据你的言行实时增减，直接决定她对你的态度是警惕、顺从还是极度依赖。
3. 桌面级空间感知与全域物理互动
全局视线追踪：无论你的鼠标在屏幕的哪个角落，她的视线都会跟随你（基于后台透明窗体全局坐标映射）。
实体防线与触碰感应：支持头部、脸颊、手臂、胸部等多个部位的点击碰撞检测，会立刻打断她当前的动作，并触发截然不同的娇羞或抗拒反馈。
4. 现实生物钟与自动化换装系统
她拥有与现实世界完全同步的时间感知。
白天她可能会穿着紧身制服或连衣裙；到了晚上，她会自动换上轻薄的睡衣；如果是深夜且你们的亲密度足够高，你甚至可能会碰见她刚洗完澡裹着浴巾的慵懒模样。特定节假日（如元旦、圣诞节）还会自动换上民族风斗篷等特殊服饰。
5. 极度细腻的“发呆与独处”机制
当你长时间不操作电脑或离开时，她会逐渐移开视线进入“发呆”状态。
根据离开的时间长短（3分钟、5分钟、甚至更久），系统会在后台自动唤醒大脑，让她小声碎碎念、陷入沉思，甚至因为长时间未互动而触发“浴巾意外滑落”等极低概率的隐藏物理彩蛋。
6. 专业的开发者物理调试引擎
除了主陪伴程序，项目还附带了“终极系统测试控制台”，支持 116 项 Live2D 参数全量直调、时空强制跃迁测试、动作序列剧本推演以及 API 嗅探探针，方便开发者进行极致的二次开发。

## 【安装与环境配置指南】

第一步：准备运行环境
请确保你的电脑上已经安装了 Python 3.8 或更高版本。

第二步：一键安装核心依赖项
打开命令行终端，进入项目所在的根目录，运行以下命令安装所有需要的第三方库：
pip install -r requirements.txt

第三步：模型前端资源准备 (极其重要)
请确保你的项目根目录中包含一个名为 web 的文件夹，里面必须存放着 index.html 以及 Romasha 的 Live2D 核心模型文件（如 .moc3、cdi3.json、材质贴图等）。后端的 PyQt5 会通过 QWebEngineView 自动加载这个本地网页来渲染整个物理引擎。

第四步：配置文件设置 (config.json)
首次运行程序时，如果目录下没有 config.json，系统会在控制台提示并自动生成一个默认的配置文件。你需要打开它并填入你自己的 API 信息：

* api_key：你的大模型 API 密钥。
* base_url：大模型接口地址（支持任何 OpenAI 兼容格式的转发地址）。
* target_model：你想要调用的具体模型名称。

(注意：此文件还会自动记录程序退出时的窗口坐标、缩放比例以及当前的亲密度，请勿随意破坏内部的 JSON 格式。)

## 【启动与日常使用说明】

启动主陪伴程序：
在命令行运行：python main.py
(由于后台需要预热本地向量数据库和嵌入模型，启动时控制台可能会停留几秒，出现“记忆库唤醒成功”的提示后，Romasha 就会以透明无边框的形态出现在你的桌面上。)

内置快捷指令
在与她聊天的输入框（右键点击出现，左键拖拽互动等）中直接输入并发送即可：

/reset：触发不可逆的记忆重置，清空所有长时记忆，亲密度归零，回到初见状态。

/track 0：关闭鼠标视线追踪（让她不再盯着你的鼠标看）。

/track 1：重新开启鼠标视线追踪。

/touch 0：关闭物理触碰感应屏障（点击模型不再触发触摸反馈）。

/touch 1：恢复物理触碰感应。

## 【开发者与测试工具套件】

如果你想对模型进行深度调试、修复动作映射或提取记忆档案，可以使用以下内置的独立测试工具：

1. 终极系统测试控制台 (test_system_engine.py)
运行命令：python test_system_engine.py
这是一个极其强大的带图形界面的沙盒引擎。你可以在这里不消耗真实亲密度的情况下，测试所有的换装、面部红晕、时空跳转，甚至可以查看到底组装了怎样的 System Prompt 发送给了大模型（包含 Token 消耗预估探针）。
2. 记忆读取与导出工具 (view_memories.py)
运行命令：python view_memories.py
运行后，系统会将 ChromaDB 数据库中存留的你们所有的过往回忆提取出来，并按时间顺序生成一份名为 romasha_memories_export.txt 的可读文本档案，方便你回顾或做剧情备份。
3. 纯动作与服装映射测试面板 (test_motions.py)
运行命令：python test_motions.py
这是一个轻量级的 Live2D 动作检视器，专门用来快速预览 motion_manager.py 中设定的所有动作和 outfit_manager.py 中的服装切换是否渲染正常。
